Security Architecture
🔒

Encryption At Rest

Every user question is encrypted using AES-256 (Fernet) before being written to the database. Even if someone accesses the DB file directly, they see unreadable ciphertext.

cryptography.fernet
🌐

Encryption In Transit

All data between user browser and server travels over HTTPS (TLS 1.3). Enabled automatically on Railway / Render / any cloud host. Never deploy on plain HTTP.

TLS 1.3
🗝️

Key Management

The ENCRYPTION_KEY lives only in the .env file, never in code. The .env is gitignored. Without this key, the ciphertext in the DB is permanently unreadable.

.env + .gitignore
👤

Admin-Only Decryption

decrypt_text() is called only inside the /admin/chats route. The admin panel is password protected. Regular users never trigger decryption — ever.

Route-level control
🛡️

Password Hashing

Admin passwords stored as bcrypt hashes, never plain text. Even if DB is leaked, passwords cannot be reversed. Salt is automatically applied per hash.

bcrypt
🚦

Rate Limiting

FastAPI middleware limits requests per IP to prevent spam and API cost abuse. Stops bots from draining your Claude API credits or flooding the database.

slowapi
